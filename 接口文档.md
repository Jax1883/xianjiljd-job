# 西安吉利京东工具 - 接口文档

**服务地址：** `http://{host}:8380`  
**版本：** 1.0.0  
**更新时间：** 2026-05-07（v1.1 新增分页查询）  

---

## 概述

本服务提供出库单回传和收货单回传的查询与激活功能，供外部系统对接使用。

- 所有接口均使用 `Content-Type: application/json`
- 无需鉴权
- 响应格式统一为 JSON

---

## 通用响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | int | 业务状态码：200=成功，400=参数错误，500=服务器错误 |
| `message` | String | 响应消息 |
| `data` | Object/Array/null | 响应数据 |

## 接口列表

### 接口一：查询回传状态

**功能：** 查询出库单或收货单的同步回传状态。

**请求方式：** `POST`  
**接口路径：** `/tools/xianjilijd/queryOrderBack`  
**完整地址：** `http://{host}:8380/tools/xianjilijd/queryOrderBack`

#### 请求头

```
Content-Type: application/json
```

#### 请求参数

| 参数名 | 类型 | 是否必填 | 说明 |
|--------|------|---------|------|
| `orderType` | String | 是 | 订单类型，枚举值：`入库单回传` 或 `出库单回传` |
| `code` | String 或 List\<String\> | 否 | 单据编码过滤。支持逗号分隔字符串或 JSON 数组，最多 500 个；不传则查询该类型全量数据 |
| `pageNum` | int | 否 | 页码，从 1 开始，默认 `1` |
| `pageSize` | int | 否 | 每页条数，默认 `10` |

> **注意：**
> - `code` 为可选参数，不传时返回该 `orderType` 下的全量分页数据
> - `code` 最多传 500 个，超出返回 400 错误，自动去重和过滤空值

#### 请求示例

**查询全量（不传 code，取第一页默认 10 条）**
```json
{
  "orderType": "出库单回传"
}
```

**指定页码和每页条数**
```json
{
  "orderType": "出库单回传",
  "pageNum": 2,
  "pageSize": 20
}
```



**JSON 数组格式**
```json
{
  "orderType": "入库单回传",
  "code": ["HKCK6105230603000002", "HKCK6105230603000003"],
  "pageNum": 1,
  "pageSize": 10
}
```

#### 响应参数

`data` 为分页对象：

| 字段 | 类型 | 说明 |
|------|------|------|
| `total` | Long | 符合条件的总记录数 |
| `pageNum` | Long | 当前页码 |
| `pageSize` | Long | 每页条数 |
| `records` | List | 当前页数据列表 |

`records` 中每条记录字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | Long | 数据库记录 ID |
| `responseCode` | String | 单据编码（出库单编码或收货单编码） |
| `isSync` | Integer | 同步状态：`0`=未同步（或已重置），`1`=已同步 |
| `syncTimes` | Integer | 同步次数 |
| `lastModifyDate` | String | 最后修改时间，格式：`yyyy-MM-dd HH:mm:ss` |
| `orderType` | String | 订单类型：`入库单回传` 或 `出库单回传` |

#### 响应示例

**查询到数据（200）**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 100,
    "pageNum": 1,
    "pageSize": 10,
    "records": [
      {
        "id": 12345,
        "responseCode": "HKCK6105230603000002",
        "isSync": 1,
        "syncTimes": 3,
        "lastModifyDate": "2026-04-30 10:11:22",
        "orderType": "出库单回传"
      },
      {
        "id": 12346,
        "responseCode": "HKCK6105230603000003",
        "isSync": 0,
        "syncTimes": 0,
        "lastModifyDate": "2026-04-30 11:05:33",
        "orderType": "出库单回传"
      }
    ]
  }
}
```

**无匹配数据（200）**
```json
{
  "code": 200,
  "message": "无数据，请人工核实",
  "data": {
    "total": 0,
    "pageNum": 1,
    "pageSize": 10,
    "records": []
  }
}
```

**参数错误（400）**
```json
{
  "code": 400,
  "message": "orderType 必须是 入库单回传 或 出库单回传",
  "data": null
}
```

### 接口二：激活回传（重置同步状态）

**功能：** 将指定记录的同步状态和同步次数重置为 0，触发系统重新同步。

**请求方式：** `POST`  
**接口路径：** `/tools/xianjilijd/activateOrderBack`  
**完整地址：** `http://{host}:8380/tools/xianjilijd/activateOrderBack`

#### 请求头

```
Content-Type: application/json
```

#### 请求参数

| 参数名 | 类型 | 是否必填 | 最大数量 | 说明 |
|--------|------|---------|---------|------|
| `orderType` | String | 是 | - | 订单类型，枚举值：`入库单回传` 或 `出库单回传` |
| `requestIds` | String 或 List\<Long\> | 是 | 1000 | 数据库记录 ID（即查询接口返回的 `id` 字段）。支持两种格式：逗号分隔的字符串，或 JSON Long 数组 |

> **注意：**
> - `requestIds` 最多传 1000 个，超出返回 400 错误
> - `requestIds` 自动去重，自动过滤无效值
> - 该操作使用事务，任何异常都会自动回滚，保证数据一致性
> - 仅修改 `is_sync` 和 `sync_times` 两个字段，其余字段不受影响

#### 请求示例

**格式：JSON 数组**

```json
{
  "orderType": "入库单回传",
  "requestIds": [12345, 12346, 12347]
}
```



#### 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| `data.updated` | Integer | 实际更新的记录数（若传入的 ID 在数据库中不存在，则不计入） |

#### 响应示例

**激活成功（200）**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "updated": 3
  }
}
```

**参数错误（400）**
```json
{
  "code": 400,
  "message": "requestIds 不能为空",
  "data": null
}
```

**服务器错误（500）**
```json
{
  "code": 500,
  "message": "服务器内部错误",
  "data": null
}
```

## 错误码一览

| code | HTTP 状态码 | 含义 |
|------|-----------|------|
| 200 | 200 | 请求成功 |
| 400 | 400 | 参数错误（详见 message） |
| 500 | 500 | 服务器内部错误 |

### 常见 400 错误消息

| message | 原因 |
|---------|------|
| `orderType 不能为空` | 未传 orderType 或值为空 |
| `orderType 必须是 入库单回传 或 出库单回传` | orderType 不在枚举范围内 |
| `code 数量超过上限 500` | code 列表超过 500 个 |
| `requestIds 不能为空` | 未传 requestIds 或值为空/全无效 |
| `requestIds 数量超过上限 1000` | requestIds 列表超过 1000 个 |
| `参数校验失败` | 参数格式不符合要求 |
| `请求体格式错误，请检查 JSON 是否合法` | 请求体 JSON 解析失败 |

## 典型对接流程

**场景：某单据未能回传，需要手动触发重新同步**

```
步骤1. 调用查询接口，传入单据编码，确认 isSync 状态
步骤2. 记录返回的 id 字段
步骤3. 调用激活接口，传入对应的 id，将同步状态重置为 0
步骤4. 等待系统后台任务自动重新同步
步骤5. 再次调用查询接口，确认 isSync 已变为 1
```

---

## 数据说明

### isSync 字段含义

| 值 | 含义 |
|----|------|
| `0` | 未同步（初始状态或已重置） |
| `1` | 已同步成功 |

### orderType 与数据表对应关系

| orderType 值 | 对应数据表 | 编码字段 |
|-------------|-----------|---------|
| `入库单回传` | `receipt_note_back` | `receipt_code` |
| `出库单回传` | `out_order_back` | `out_order_code` |
